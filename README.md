# Real-Time Stock Analytics Dashboard

A full end-to-end data engineering project that simulates live stock price streams using **Apache Kafka**, processes them in real-time and batch with **Apache Spark**, and visualizes everything on a live **Flask + WebSocket dashboard**.

---

## Architecture
Kafka Producer
      ↓
Spark Streaming (Real - time processing) → results.json
      ↓
CSV Storage (intermediate data) → stock_data.csv
      ↓
Spark Batch (ML + analytics every 10s) → batch_results.json
      ↓
Flask API
      ↓
Live Dashboard (Socket.IO)

---

## Features

### Real-Time Stream (spark_stream.py)
- **Live price statistics per stock** — average, min, max, tick count, standard deviation
- **Coefficient of Variation (CV)** — normalized volatility as % of mean
- **Z-score** — how many standard deviations the current price is from the session mean
- **Anomaly detection** — flagged when |z-score| > 2.0
- **Price momentum** — current price minus session average
- **Trend label** — UPTREND / DOWNTREND / SIDEWAYS based on momentum threshold
- Writes every raw tick to `stock_data.csv` for batch processing

### Batch Analytics (spark_batch.py)
- **Descriptive statistics** — mean, stddev, min, max, skewness, kurtosis, P25/P50/P75/P95
- **Linear Regression** — next price prediction with R², RMSE
- **Bollinger Bands** — upper/lower bounds (mean ± 2σ) with breakout detection (BREAKOUT_UP / WITHIN_BANDS / BREAKOUT_DOWN)
- **Trend classification** — slope of last 20 ticks → UPTREND / DOWNTREND / SIDEWAYS

### Live Dashboard (index.html + app.py)
- **KPI row**: total messages, no. of stocks, uptime, anamoly
- **Top 3 movers** cards (sorted by momentum) with trend tags, Z-score, Bollinger signal, and ML predicted next price
- **Single-stock price chart** with a dropdown selector and Bollinger Band overlays
- **ML predictions panel** — current vs. predicted price with R² and RMSE
- **Bollinger Bands panel** — band range and breakout signal per stock
- **Descriptive statistics panel** — tabbed per stock with 12 metrics
- **Live alerts feed** — session highs/lows, anomaly triggers, Bollinger breakouts

## Prerequisites

Make sure the following are installed on your machine before starting.

- Python 3.10.5
- Java (JDK 17)
- Apache Kafka 2.13-4.2.0
- Apache Spark 3.5.8 (Pre-build for Apache Hadoop 3.3 & later)


### Python packages

```bash
pip install kafka-python pyspark flask flask-socketio pandas
```

### Kafka–Spark connector JAR

Spark needs the Kafka connector JAR to read from Kafka. Download and place it in your Spark `jars/` folder, or pass it at runtime via `--packages`:

```
org.apache.spark:spark-sql-kafka-0-10_2.12:3.3.0
```

If you use `spark-submit`, add:
```bash
--packages org.apache.spark:spark-sql-kafka-0-10_2.12:3.3.0
```

## Running the Project

The project requires **4 separate terminals** running in parallel.

### Terminal 1 — Start Kafka Broker

```bash
# From your Kafka installation directory
.\bin\windows\kafka-server-start.bat .\config\server.properties

```

### Terminal 2 — Create the Kafka topic (first time only)

```bash
bin/kafka-topics.sh --create \
  --topic stock-data \
  --bootstrap-server localhost:9092 \
  --partitions 1 \
  --replication-factor 1
```

You can verify it was created:

```bash
bin/kafka-topics.sh --list --bootstrap-server localhost:9092
```

### Terminal 3 — Start the Kafka producer

```bash
python producer.py
```

You should see output like:
```
Producer started. Sending stock data to Kafka...
Sent: {'stock': 'APPLE', 'price': 176.23, 'timestamp': 1718000000.0}
```

### Terminal 4 — Start Spark Streaming

```bash
spark-submit \
  --packages org.apache.spark:spark-sql-kafka-0-10_2.13:3.5.8 \
  spark_stream.py
```

Wait until you see:
```
[Spark] Streaming started.
  Live stats  → /path/to/results.json
  Raw ticks   → /path/to/stock_data.csv
```

### Terminal 5 — Start Spark Batch (optional but recommended)

Run this after `spark_stream.py` has collected some data (at least 10 ticks per stock):

```bash
python spark_batch.py
```

This runs every 10 seconds automatically and writes `batch_results.json`.

### Terminal 6 — Start the Flask dashboard

```bash
python app.py
```

Then open your browser at:

```
http://localhost:5000
```

## Configuration

### Change stocks or starting prices

In `producer.py`, edit the `stocks` list and `prices` dictionary:

```python
stocks = ["APPLE", "GOOGLE", "TESLA", "MICROSOFT", "AMAZON", "NETFLIX", "META"]
prices = {
    "APPLE": 175.0,
    ...
}
```

### Change price movement range

```python
change = random.uniform(-6.5, 8.5)   # min/max price change per tick
```

### Change producer send frequency

```python
time.sleep(0.8)   # seconds between each price tick
```

### Change Spark batch interval

In `spark_stream.py`:
```python
.trigger(processingTime="3 seconds")   # how often Spark processes a batch
```

In `spark_batch.py`:
```python
RUN_INTERVAL = 10   # seconds between batch analytics runs
```

### Change Kafka broker address

All files default to `localhost:9092`. To change, update this line in `producer.py`, `spark_stream.py`, and `spark_batch.py`:

```python
bootstrap_servers='localhost:9092'   # producer.py
.option("kafka.bootstrap.servers", "localhost:9092")   # spark files
```

## How Each File Works

### `producer.py`
Simulates a live market feed by randomly walking prices for 7 stocks and publishing JSON messages to the `stock-data` Kafka topic every 0.8 seconds. Each message contains `stock`, `price`, and `timestamp`.

### `spark_stream.py`
Reads the Kafka stream in real-time using Spark Structured Streaming. Runs two parallel queries:
- **Stream 1** appends every raw price tick to `stock_data.csv` (used by batch)
- **Stream 2** computes aggregated per-stock stats (avg, min, max, stddev, Z-score, trend, etc.) and writes them to `results.json` every 3 seconds

### `spark_batch.py`
Reads the full `stock_data.csv` history every 10 seconds and runs 5 computations: descriptive stats, linear regression price prediction, Bollinger Bands, trend classification (slope of last 20 ticks), and a pairwise correlation matrix. Writes all results to `batch_results.json`.

### `app.py`
Flask server with SocketIO. A background thread reads `results.json` and `batch_results.json` every 2 seconds and pushes the merged payload to all connected browser clients over WebSocket. Also exposes `/api/data` and `/api/batch` REST endpoints as fallbacks.

### `templates/index.html`
Pure HTML/CSS/JS dashboard. Connects to Flask via Socket.IO and updates all panels on every `update` event. Uses Chart.js for the price chart and sparklines. No external UI framework — everything is hand-crafted.

## Troubleshooting

**`ModuleNotFoundError: No module named 'kafka'`**
```bash
pip install kafka-python
```

**`java.lang.ClassNotFoundException` in Spark**
The Kafka connector JAR is missing. Add `--packages org.apache.spark:spark-sql-kafka-0-10_2.12:3.3.0` to your `spark-submit` command or set `PYSPARK_SUBMIT_ARGS` in your environment.

**Dashboard shows "Waiting for data…" indefinitely**
- Check that `producer.py` is running and sending messages
- Check that `spark_stream.py` is running and has written `results.json`
- Confirm `results.json` exists in the same directory as `app.py`
- Check browser console for WebSocket errors

**`batch_results.json` never appears**
- `spark_batch.py` needs at least 10 ticks per stock before it trains the model
- Make sure `stock_data.csv` exists (created by `spark_stream.py`)
- Run `spark_batch.py` after the stream has been running for at least 30 seconds

**Port 5000 already in use**
```bash
# Change the port in app.py
socketio.run(app, host="0.0.0.0", port=5001, ...)
```

**Kafka topic already exists error**
Safe to ignore — the topic is already there from a previous run. The producer will connect normally.
